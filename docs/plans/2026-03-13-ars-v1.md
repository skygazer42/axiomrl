# ARS V1 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a narrow but honest `ARS` baseline for vector-observation, continuous-action control by training a deterministic MLP policy with mirrored parameter perturbations and reward-ranked updates.

**Architecture:** Keep the release deliberately small and aligned with the current continuous-control package surface. Reuse the existing `TrainConfig`, run directories, checkpointing, evaluation/prediction workflow, action scaling, and managed API, but add a dedicated `ARS` model, learner, and trainer built around synchronous rollout evaluation of positive and negative parameter perturbations. Explicitly do not implement parallel workers, observation normalization across processes, discrete-action ARS, or multi-process search orchestration in this batch.

**Tech Stack:** Python 3.10, PyTorch, Gymnasium, pytest, existing `rl_training` experiment/runtime infrastructure.
